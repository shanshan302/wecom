# 企业微信 AI / Agent 情报日报 - 2026-07-09

## 今日结论

暂无发现 2026-07-09 当天有“企业微信允许 AI Agent 更广泛地直接向外部联系人、客户群或任意会话主动发消息”的官方新证据。

今天最值得记录的变化是腾讯云 ADP《将应用发布到企微智能机器人》文档已在 2026-07-03 更新，明确把“长连接 WebSocket + 扫码绑定”列为推荐接入方式，并写清机器人会话场景和限制：群里 @机器人、单聊机器人；机器人需在 3 分钟内回答；同一机器人同一时间只能保持一种接入方式，长连接只能保持一个有效连接。这强化了“企微智能机器人是 Agent 入口”的确定性，但没有改变外部联系人、客户群、客户联系群发的权限模型。

同时，`WeComTeam/wecom-cli` 在 2026-07-08 新增 issue #90，直接询问“现在是不是只能给同事发消息，不能给客户发消息？”。这是一个值得跟进的社区/生态信号：开发者仍在确认客户消息边界，尚无维护者回复，不能视为能力开放。

## 消息发送与 API 开放迹象

- [已确认][官方] ADP/企微智能机器人可作为 Agent 发布渠道；可在机器人单聊、群里 @机器人时收发消息并支持流式效果；长连接模式推荐且可扫码绑定。发送主体：企微智能机器人/ADP 应用；发送目标：机器人单聊或群聊会话上下文；发送模式：会话内回复、流式回复；权限模型：企业后台机器人、企业 ID、Bot ID、Secret 或扫码授权、接入方式约束；运营限制：3 分钟回答、一个应用一个机器人、一个有效长连接。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [已确认][官方] ADP 2026-06 产品动态中的“定时任务执行结果推送到微信或企业微信机器人”是主动结果回传信号，但目标仍是微信/企业微信机器人。不能推断为可向客户、客户群或任意会话自由外发。来源：[腾讯云 ADP 产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [已确认][官方/开发者文档] 企业微信客户联系“创建企业群发”仍是创建群发任务并通知成员发送给客户或客户群；调用接口不等于直接发送，通常仍涉及成员确认、客户联系群发权限和频控。影响：客户/客户群触达仍是任务+成员发送模型，不是 Agent 直接发消息模型。来源：[企业微信开发者中心：创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [已确认][官方/开发者文档] 企业微信自建应用向群聊会话发消息的接口仍主要面向企业内部群；公开文档和腾讯云开发者社区整理均强调应用消息不支持添加企业外部联系人进群，暂时仅企业内自建应用接入。影响：内部群通知和外部联系人/客户群触达仍是不同权限域。来源：[企业微信开发者中心：发送消息到群聊会话](https://developer.work.weixin.qq.com/document/path/90244)、[腾讯云开发者社区：接入企业微信通知方案](https://cloud.tencent.com/developer/article/2571033)
- [推断][半官方/生态] `wecom-openclaw-plugin` 的 proactive messaging 更可能是基于 Bot/Agent 双模式、企业内部用户/群/部门/标签、自建应用回退投递及策略控制的组合能力，而非客户联系 API 的新开放。理由：README 同时强调 DM/Group policy、access control、Bot/Agent mode；官方客户联系群发文档未出现 Agent 直发客户/客户群的新说明。来源：[wecom-openclaw-plugin](https://github.com/WeComTeam/wecom-openclaw-plugin)、[企业微信群发客户/客户群](https://developer.work.weixin.qq.com/document/path/92135)

影响拆解：

- 发送主体：确认有企微智能机器人、企业微信自建应用、群机器人、客户联系群发任务、ADP 应用发布渠道；未确认有“任意 AI Agent 身份”。
- 发送目标：机器人单聊/群聊、内部用户/内部群、部门/标签、自建应用可见范围、微信客服入口、客户/客户群群发任务仍属不同通道。
- 发送模式：机器人通道增强为长连接、扫码绑定、流式回复、定时任务结果回传；客户联系仍是群发任务和成员执行。
- 权限模型：Bot ID/Secret、扫码授权、企业 ID、可见范围、一个有效长连接、客户联系权限、成员确认和频控仍是核心约束。
- 运营限制：未发现客户群、外部联系人频控、成员确认、客户联系权限被取消或放宽。
