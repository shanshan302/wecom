# 企业微信 AI / Agent 情报日报 - 2026-07-04

## 今日结论

暂无发现“企业微信允许更广泛 Agent 直接向外部联系人/客户群主动发消息”的可信新证据。

今天最值得关注的变化，是腾讯云 ADP《将应用发布到企微智能机器人》在 `2026-07-03 11:33:01` 更新后，明确把企微智能机器人接入拆成 `长连接 WebSocket（推荐）` 与 `URL 回调 Webhook` 两种方式，并写清使用场景仍是“群里 @智能机器人”和“单聊智能机器人”。这增强了 Agent 接入企业微信的工程可用性，但没有改变外部客户触达、客户群触达、成员确认式群发等公开权限边界。来源：[腾讯云 ADP 文档](https://cloud.tencent.com/document/product/1759/121473)

## 官方与高可信更新

- [半官方/生态][更新时间 2026-07-03] 腾讯云 ADP 更新《将应用发布到企微智能机器人》：文档称机器人支持流式回复；推荐长连接 WebSocket 模式，理由是无需公网域名/IP、无需 Nginx；URL 回调模式仍要求备案域名和 Nginx 转发。影响：接入方式更清晰，发送主体仍是企微智能机器人，不是任意 Agent 身份。来源：[将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [半官方/生态][更新时间 2026-07-03] 同一文档列出限制：一个应用只能发布到一个智能机器人；机器人只能在用户发消息后的 3 分钟内回答，超时截断；同一机器人同一时间只能保持一种接入方式；长连接模式每个机器人同一时间只能保持一个有效长连接。影响：操作限制变得更明确，没有放宽目标会话或外部客户发送权限。来源：[腾讯云 ADP 文档](https://cloud.tencent.com/document/product/1759/121473)
- [官方][最后更新 2026-05-25] 企业微信开发者中心《智能机器人长连接》仍是判断机器人能力边界的核心文档：回调里区分 `single` 单聊与 `group` 群聊，`chatid` 在群聊类型返回；长连接支持主动推送流式刷新消息，并提供 `aibot_send_msg`，其中 `chatid` 支持单聊和群聊，消息类型包括 `markdown`、`template_card`、文件、图片、语音、视频等。影响：这是“机器人会话内”的主动推送能力，不等同于外部联系人/客户群常规主动触达。来源：[企业微信开发者中心：智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [半官方/生态][更新时间 2026-04-15] ADP《将应用发布到企业微信应用》仍要求企业微信应用路径配置 `AgentId / Secret / 企业ID / URL / Token / EncodingAESKey / 企业可信 IP`，并通过企业微信工作台应用进入聊天。影响：企业微信应用路径仍是传统应用权限与可信 IP 模型。来源：[将应用发布到企业微信应用](https://cloud.tencent.com/document/product/1759/119175)

## 消息发送与 API 开放迹象

- [已确认][官方][最后更新 2026-05-25] 企业微信智能机器人长连接支持在机器人单聊/群聊上下文内发送或刷新消息，`chatid` 可按单聊用户 userid 或群聊回调获得的 chatid 解析。影响：发送目标是机器人已参与或已建立上下文的单聊/群聊；没有证据表明它可绕过客户联系、客户群或外部联系人的既有限制。来源：[智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [已确认][半官方/生态][更新时间 2026-07-03] ADP 企微智能机器人渠道的公开场景仍只有“群里 @智能机器人”和“用户单聊智能机器人”；返回限制为每次最多 2048 字节。影响：发送模式偏回复/会话内流式输出，不是批量或任意外发。来源：[ADP 企微智能机器人文档](https://cloud.tencent.com/document/product/1759/121473)
- [已确认][半官方/生态][访问日期 2026-07-04] 企业微信客户联系公开镜像中的“创建企业群发”仍是创建任务并通知成员发送，不是企业或 Agent 直接发送给客户/客户群。影响：客户群发仍偏成员确认式触达。来源：[Apifox 镜像：创建企业群发](https://qiyeweixin.apifox.cn/api-10061297)
- [已确认][半官方/生态][访问日期 2026-07-04] “发送新客户欢迎语”仍依赖 `welcome_code`，属于新客户事件触发型消息；“微信客服发送消息”仍围绕客服会话窗口。影响：客户侧发送仍受事件、时窗、次数等规则约束。来源：[Apifox 镜像：发送新客户欢迎语](https://qiyeweixin.apifox.cn/api-10061298)、[Apifox 镜像：微信客服发送消息](https://qiyeweixin.apifox.cn/api-10061328)

影响拆解：

- 发送主体：新增清晰度集中在 `企微智能机器人`；企业微信应用仍走 `AgentId / Secret / 可信 IP / 回调` 模型。
- 发送目标：机器人单聊和机器人所在群聊最清晰；外部联系人、客户群仍没有看到放宽证据。
- 发送模式：长连接增强了流式回复和主动推送刷新；客户联系侧仍多为成员确认、事件触发或客服窗口。
- 权限模型：`Bot ID / Secret`、扫码绑定、URL 回调、可信 IP、企业主体域名等限制继续存在。
- 运营限制：3 分钟回复窗、单机器人单有效长连接、一个应用一个机器人、2048 字节返回限制需要纳入方案设计。
