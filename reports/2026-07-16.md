# 企业微信 AI / Agent 情报日报 - 2026-07-16

## 今日结论

暂无重大变化：截至 2026-07-16，未发现企业微信官方扩大 AI Agent 对外部联系人、微信客户或客户群的任意主动消息权限。

本次最有价值的边界信号有两项：其一，企业微信团队维护的 `wecom-openclaw-plugin` README 明确列出 Bot/自建应用双模式、MCP 工具、流式回复，以及向指定用户、群、部门或标签主动发送；但仓库最新提交仍停留在 2026-07-02，且“用户/群/部门/标签”应按企业内部通讯录和自建应用权限理解，不能外推为微信客户或客户群。其二，腾讯云 IM Desk 已于 2026-07-01 文档化 ADP 智能体一键接入在线客服，改变的是 Agent 的承接渠道和集成方式，不是企业微信客户联系 API。

- [官方][最近更新 2026-06-15；检索 2026-07-16] ADP 产品概述继续确认 Claw、标准、多智能体与 Workflow 模式，内置连接器/工具并支持 MCP，且可发布至企业微信等渠道。“发布到企微”仍是渠道接入，不等于客户联系授权。来源：[腾讯云 ADP 产品概述](https://cloud.tencent.cn/document/product/1759/104193)
- [官方][发布 2026-06-05；检索 2026-07-16] 腾讯云 AI 产业应用大会发布 WorkBuddy 企业版，并升级 ClawPro 与 ADP，说明腾讯正在形成 Agent 构建、治理与多端使用的完整链路；未宣布企业微信客户消息权限变化。来源：[腾讯官方大会总结](https://www.tencent.com/en-us/articles/2202350.html)
- [官方][版本日期 2026-06-23；检索 2026-07-16] 企业微信 5.0.9 产品信息仍聚焦智能搜索、总结、机器人及客户服务总结；未见 Agent 代成员向微信客户/客户群自由主动发信的版本说明。来源：[企业微信产品页/版本信息](https://wwurl.rtxapp.com/)

## AI / Agent / MCP / 智能机器人动态

- [半官方/生态][仓库最新提交 2026-07-02；检索 2026-07-16] 企业微信团队维护的 `wecom-openclaw-plugin` 已公开 Bot（WebSocket/Webhook）和 Agent（自建应用）双模式，支持私聊、群聊、流式回复、主动消息、MCP 工具与 15 个内置 Skills；README 还描述 Agent HTTP API 作为 Bot WebSocket 不可用时的出站回退。仓库提交页显示 7 月 2 日后没有新提交，因此这是对既有能力边界的进一步确认，不是今日新发布。来源：[WeComTeam/wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)、[提交记录](https://github.com/WecomTeam/wecom-openclaw-plugin/commits/main/)
- [半官方/生态][检索 2026-07-16] 插件 README 的“向指定用户、群、部门或标签主动发送”是当前最强的广义主动发送信号，但它同时把 Agent 模式定义为“自建应用、API 驱动消息”。因此高可信解释是复用企业内部应用消息能力，而非开放客户联系消息。来源：[插件 README](https://github.com/WecomTeam/wecom-openclaw-plugin#readme)
- [半官方/生态][仓库观察 2026-07-16] `wecom-cli` 未发现 7 月 14 日观察点后的公开能力更新；关于能否给客户发送消息的 issue #90 仍未见维护者给出扩权结论。来源：[WeComTeam/wecom-cli](https://github.com/WecomTeam/wecom-cli)、[issue #90](https://github.com/WecomTeam/wecom-cli/issues/90)
- [社区][发布 2026-06-18；腾讯云 MCP 广场收录] `wecom-bot-mcp-server` 提供群机器人文本、Markdown、文件等发送工具，是把既有群机器人 webhook 包装成 MCP 的工程方案；腾讯云收录提升了可发现性，但作者为社区开发者，且没有改变 webhook 的目标范围和权限。来源：[腾讯云 MCP 广场：企业微信机器人消息服务器](https://developer.cloud.tencent.com/mcp/server/10854)

## 消息发送与 API 开放迹象

- [已确认][半官方/生态] **企业内部主动发送链路更清晰**：官方团队插件说明自建应用模式可向指定用户、群、部门或标签主动发送，并可在 Bot 通道不可用时回退到 Agent HTTP API。来源：[插件 README](https://github.com/WecomTeam/wecom-openclaw-plugin#readme)
- [已确认][官方] **客服 Agent 接入更简单**：腾讯云 IM Desk 可直接接入 ADP，让机器人进入客服会话流程；这是 Desk/ADP 集成能力，不是企业微信客户联系接口。来源：[腾讯云 IM Desk 接入 ADP](https://cloud.tencent.com/document/product/269/133095)
- [已确认][官方] **客户群发仍是独立权限域**：创建企业群发等客户联系接口仍要求客户联系权限、成员执行/触达规则和平台运营限制；未发现智能机器人、OpenClaw 或 ADP 可绕过这些约束。来源：[创建企业群发 API 文档镜像（含官方接口路径）](https://qiyeweixin.apifox.cn/api-10061297)
- [推断][高可信] 插件中的 `users/groups/departments/tags` 与客户联系体系中的 `external_userid/chat_id` 不是同一组目标标识；在官方客户联系文档没有同步变化前，不能把前者解读为 Agent 可向微信客户或客户群自由主动发送。

影响拆解：

- 发送主体：企微智能机器人、自建应用，或 Desk 内的 ADP 机器人；未出现独立于应用授权的通用 Agent 身份。
- 消息目标：内部用户/群/部门/标签，以及已进入 Desk 流程的客服会话；未确认扩展到任意微信客户或客户群。
- 发送模式：内部应用主动消息、机器人会话回复、流式回复、客服承接均已具备；客户触达仍走专用客户联系流程。
- 权限模型：Bot ID/Secret、回调配置、自建应用可见范围、ADP AppKey/云密钥及 Desk 会话配置共同约束；客户联系授权未被替代。
- 运营限制：未发现客户群发的频次、成员执行、客户接收或合规限制被取消或放宽。
