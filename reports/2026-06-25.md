# 企业微信 AI / Agent 情报日报 - 2026-06-25

## 今日结论

暂无证据表明企业微信今天放宽了“更广泛 API / Agent 直接主动发消息给外部联系人、客户群或任意会话”的官方边界。

今天最值得记录的新增高可信信号，主要来自腾讯云 ADP / ClawPro，而不是企业微信服务端 API 本身：

- [半官方/生态][更新页最近更新时间 2026-06-11] 腾讯云 ADP 产品动态新增了两条与企微直接相关的 2026 年 6 月能力：`智能工作台支持定时任务`，且结果可推送到“微信或企业微信机器人”；`智能工作台支持远程终端渠道`，明确支持通过“企业微信智能机器人、微信 ClawBot”远程对话。这意味着企微机器人正在被腾讯云官方进一步产品化为 Agent 结果回传和远程执行入口，但这仍是机器人通道，不是企业微信开放任意身份代发。来源：[腾讯云智能体开发平台产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [官方][访问日期 2026-06-25；页面内嵌文档时间戳对应约 2026-05-20 UTC] 企业微信帮助中心仍明确：OpenClaw 接入企微智能机器人时，`API模式（长连接）` 支持“被动回复多条消息、主动向用户发消息”，但 `暂不支持在外部群 / 客户群中使用`。这仍是今天判断消息边界的最高可信依据。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [官方][最后更新 2025-11-17] 客户联系侧官方能力仍是“创建企业群发任务并通知成员发送”，`不会直接发送消息给客户/客户群，需要成员确认后才会执行发送`。因此今天没有发现“企业直接对客户群即时主动出站”的新增证据。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)

## 官方与高可信更新

- [半官方/生态][最近更新时间 2026-06-11] 腾讯云 ADP 产品动态列出 `智能工作台支持定时任务`，并明确“将执行结果推送到微信或企业微信机器人”。这说明腾讯云正在把企微机器人定位为 Agent 任务结果的标准落点之一。发送主体仍是 ADP/ClawPro 对接的机器人，不是普通成员身份。来源：[产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [半官方/生态][最近更新时间 2026-06-11] 同一产品动态列出 `智能工作台支持远程终端渠道`，写明“支持通过企业微信智能机器人、微信 ClawBot 远程对话”。这强化了“企微机器人 = 远程 Agent 控制面”的官方生态叙事。来源：[产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [半官方/生态][最近更新时间 2026-06-11] 2026 年 5 月产品动态显示：`Claw 模式` 已成为 ADP 第四种应用模式，且 `插件广场升级为连接器与工具`，连接器被定义为可访问第三方 SaaS、企业系统与腾讯生态产品的数据与操作能力。该变化有利于 Agent 编排，不直接等于企微消息 API 放开。来源：[产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [半官方/生态][最近更新时间 2026-05-27] 腾讯云 `新建连接器` 文档仍明确：当平台预置连接器无法满足需求时，可基于 `MCP` 创建自定义连接器，并支持 `sse` 与 `streamableHttp`。这对“企微机器人 + ADP + 内部系统”的工程串联很关键，但它改变的是工具接入面，不是 WeCom 原生消息权限面。来源：[新建连接器](https://cloud.tencent.com/document/product/1759/115873)
- [半官方/生态][最近更新时间 2026-04-29] `一键部署 ClawPro` 文档明确支持企微智能机器人两种对接方式：`扫码授权` 与 `手动配置`；扫码授权流程中可“一键创建机器人后将 Bot ID、Secret 授权给智能体开发平台”。这意味着企微机器人接入门槛在腾讯云侧继续下降。来源：[一键部署 ClawPro](https://cloud.tencent.com/document/product/1759/128832)
- [半官方/生态][最近更新时间 2026-04-15] `将应用发布到企业微信应用` 文档仍然存在两种路径：简单主页方式，以及带域名、回调、可信 IP 的深度集成方式。这个入口对应的是“企业微信应用”而非“智能机器人”。来源：[将应用发布到企业微信应用](https://cloud.tencent.com/document/product/1759/119175)
- [半官方/生态][最近更新时间 2026-01-06] `将应用发布到企微智能机器人` 文档仍强调：机器人支持流式回复；使用场景是“群里 @机器人”或“单聊机器人”；限制为“一个应用只能发布到一个智能机器人”，且用户消息后机器人仅能在 `3 分钟内` 回答。来源：[将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [官方][最后更新 2025-09-24] `发送应用消息` 文档当前投递对象仍是 `touser`、`toparty`、`totag`。今天未见外部联系人、客户群、任意会话等新增目标。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)
- [官方][最后更新 2025-11-17] `创建企业群发` 文档继续明确：群发任务由企业/第三方应用创建，但不会直接发送，需要成员确认；并且仅推送给“最后跟客户进行聊天互动的企业成员”。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [官方][访问日期 2026-06-25；页面内嵌文档时间戳约 2026-05-20 UTC] 企业微信帮助中心 `如何获取 Bot ID 和 Secret` 仍说明：需要以 `API 模式 + 长连接` 创建智能机器人，随后自动展示 `Bot ID` 与 `Secret`。这继续表明当前官方 Agent/机器人生态围绕专门机器人凭证运转。来源：[如何获取 Bot ID 和 Secret](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21677)

## 消息发送与 API 开放迹象

- [已确认][官方][最后更新 2025-09-24] 企业内部消息发送模型没有变：`发送应用消息` 仍面向 `用户 / 部门 / 标签`。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)
- [已确认][官方][最后更新 2025-11-17] 客户侧触达没有变：`创建企业群发` 仍是“建任务 -> 通知成员 -> 成员确认 -> 发送”，不是 API 直接代表企业向客户/客户群即时发送。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [已确认][官方][访问日期 2026-06-25] OpenClaw 机器人通道仍支持“主动向用户发消息”，但帮助页同时写明 `暂不支持在外部群 / 客户群中使用`。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [已确认][半官方/生态][最近更新时间 2026-06-11] 腾讯云 ADP 现已支持“将定时任务执行结果推送到企业微信机器人”。这会提高机器人主动回传的可运营性，但发送主体仍是机器人/ClawPro 通道。来源：[产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [推断][基于上述官方与半官方材料综合判断] 今天没有看到任何可信新增证据表明：企业微信已允许更广泛 API 或 Agent 直接主动向 `外部联系人`、`客户群`、或 `任意员工会话` 发送消息。

影响拆解：

- 发送主体：仍是 `自建应用`、`智能机器人`、`成员确认后的客户联系群发任务`。
- 发送目标：内部消息仍限 `用户/部门/标签`；客户侧仍是 `客户/客户群群发任务`；OpenClaw 机器人仍不进 `外部群/客户群`。
- 发送模式：机器人通道支持被动回复与主动消息；ADP 现支持定时任务结果推送到企微机器人；客户联系仍是任务式/确认式。
- 权限模型：继续依赖 `Bot ID / Secret`、应用 `Secret`、可信 IP / 域名、成员可见范围、长连接权限与扫码授权。
- 运营限制：客户/客户群群发当前仍按“当月天数”控制接收上限；企微智能机器人发布到 ADP 时仍受 3 分钟应答窗口约束。
