# 企业微信 AI / Agent 情报日报 - 2026-06-28

## 今日结论

暂无重大变化。

今天没有查到企业微信在 `2026-06-28` 前后新放宽“更广泛 API / Agent 直接主动发消息给外部联系人、客户群或任意会话”的官方证据。高可信信号仍集中在两条主线：

- [官方][访问日期 2026-06-28] 企业微信帮助中心关于 OpenClaw 接入智能机器人的说明仍把能力边界放在 `API 模式机器人` 上：支持被动回复、多条回复、主动向用户发消息，但 `暂不支持在外部群 / 客户群中使用`。这依然是今天判断“消息边界是否放开”的最高可信依据。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help?doc_id=21657)
- [半官方/生态][最近更新时间 2026-06-11] 腾讯云 ADP 近一期产品动态依旧把企业微信定位为 Agent 的标准分发和执行入口之一，包括 `企业微信智能机器人渠道扫码授权方式对接`、`定时任务结果推送到企业微信机器人`、`远程终端渠道支持企业微信智能机器人`。这说明生态在强化“机器人作为 Agent 入口/出口”，但不等于服务端 API 对消息目标和主体做了更宽开放。来源：[腾讯云智能体开发平台产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [官方][GitHub API 访问日期 2026-06-28] `WecomTeam/wecom-cli` 与 `WecomTeam/wecom-openclaw-plugin` 仓库仍是企业微信团队公开推进 Agent 工具链的核心阵地；仓库元数据显示前者 `updated_at = 2026-06-27T04:55:10Z`，后者 `updated_at = 2026-06-24T13:12:03Z`。今天没看到把消息权限边界进一步推向外部联系人/客户群的新公开说明。来源：[wecom-cli](https://github.com/WecomTeam/wecom-cli)、[wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)

## 消息发送与 API 开放迹象

- [已确认][官方][访问日期 2026-06-28] 企业微信帮助中心关于 OpenClaw 机器人接入的官方页面，仍把“主动发消息”绑定在 `API 模式智能机器人` 上，而不是把普通成员会话、外部联系人、客户群都开放给 Agent。来源：[帮助中心](https://open.work.weixin.qq.com/help?doc_id=21657)
- [已确认][官方][访问日期 2026-06-28] 同一帮助页继续写明 `暂不支持在外部群 / 客户群中使用`。因此今天没有证据表明企业微信已经允许 OpenClaw / Agent 在客户群或外部群中做更普遍主动触达。来源：[帮助中心](https://open.work.weixin.qq.com/help?doc_id=21657)
- [已确认][半官方/生态][搜索结果发布时间 2 个月前] 腾讯云 ADP 文档《将应用发布到企业微信应用》仍存在，说明“企业微信应用”路径是正式渠道之一；但它属于应用接入，不自动推导出新的“任意会话主动消息”权限。来源：[文档](https://cloud.tencent.com/document/product/1759/119175)
- [已确认][半官方/生态][产品动态发布时间 2026-03] 腾讯云 ADP 产品动态中的企微相关新增，聚焦 `扫码授权对接企业微信智能机器人`、`结果推送到企业微信机器人`、`远程终端渠道`。这些改变的是接入方式、任务回传方式和操作便利性，不是消息目标范围。来源：[产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [已确认][半官方/生态][社区镜像/检索验证；访问日期 2026-06-28] 企业微信开发者文档 `创建企业群发` 标题仍在，且现有公开镜像摘要继续表明：该接口是“创建任务并通知成员发送”，`不会直接发送消息给客户/客户群，需要成员确认后才会执行发送`。因此客户联系侧今天依旧更像任务式触达，而非企业统一 Agent 即时出站。来源：[官方文档入口](https://open.work.weixin.qq.com/api/doc/90001/90143/92698)、[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061297)
- [已确认][半官方/生态][社区镜像/检索验证；访问日期 2026-06-28] `发送新客户欢迎语` 现有公开镜像摘要继续显示：依赖 `welcome_code`，需在收到事件后 `20 秒内` 调用且 `只可调用一次`。这类能力依旧是事件触发式，不是无限制主动消息。来源：[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061298)
- [推断][基于上述官方与半官方材料综合判断] 今日没有看到任何可信新增证据表明：企业微信已允许更广泛 API 或 Agent 直接主动向 `外部联系人`、`客户群`、或 `任意员工会话` 批量发送消息。

影响拆解：

- 发送主体：仍主要是 `智能机器人`、`自建应用`、`客户联系任务`。
- 发送目标：内部仍偏 `内部成员/部门/标签` 或机器人所在会话；客户侧仍是 `群发任务`、`欢迎语事件` 这类受限模式。
- 发送模式：机器人支持被动回复与有限主动消息；腾讯云 ADP 强化的是定时任务结果回传和远程终端对话。
- 权限模型：继续依赖 `Bot ID / Secret`、应用 `Secret`、成员可见范围、长连接权限、扫码授权或回调配置。
- 运营限制：外部群/客户群仍是最敏感边界，今天未见放宽迹象。
