# 企业微信 AI / Agent 情报日报 - 2026-07-10

## 今日结论

暂无发现 2026-07-10 当天有“企业微信允许 AI Agent 更广泛地直接向外部联系人、客户群或任意会话主动发消息”的官方新公告。

今天最值得跟进的信号有三类：第一，腾讯云 ADP《将应用发布到企微智能机器人》近期更新后，进一步把企微智能机器人作为 ADP 应用发布渠道讲清楚，文档明确支持长连接 WebSocket、URL 回调、单聊机器人、群里 @机器人、流式输出，但仍没有改变客户联系和客户群发的权限模型。第二，`WeComTeam/wecom-cli` 在 2026-07-08 新增 issue #90，开发者直接追问“现在是不是只能给同事发消息，不能给客户发消息？”，说明客户触达边界仍是生态痛点，但截至检索时无官方回复。第三，`WeComTeam/wecom-openclaw-plugin` 最近一次提交仍停留在 2026-07-02，增强的是企业级待办和文档表格能力，不是消息发送目标范围放宽。

结论：企微智能机器人 + ADP/OpenClaw 的 Agent 入口确定性继续增强；外部联系人、客户群、客户联系群发仍应按既有企业微信权限、成员确认和频控边界处理。

- [官方][页面搜索结果标注“last week”，检索日期 2026-07-10] 腾讯云 ADP《将应用发布到企微智能机器人》确认：ADP 应用可发布到企微智能机器人；与企业微信应用不同，机器人支持流式回复；渠道支持长连接 WebSocket 和 URL 回调两种接入方式；长连接由 ADP 服务端主动与企微建立 WebSocket，被文档列为推荐方式。影响：发送主体是 ADP 应用发布到企微智能机器人；消息目标仍是机器人单聊或群里 @机器人后的会话上下文；没有出现客户群或外部联系人直发开放。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)

## AI / Agent / MCP / 智能机器人动态

- [官方][检索日期 2026-07-10] 腾讯云 ADP 文档的效果体验明确包含两种会话：用户在企业微信客户端通讯录找到智能机器人发起单聊；群聊中选择智能机器人，并通过 @智能机器人 发起提问，支持思考过程和流式输出。影响：Agent 体验更接近会话内机器人回复，不是任意后台异步外发通道。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)

## 消息发送与 API 开放迹象

- [已确认][官方] ADP 企微智能机器人渠道支持：单聊机器人、群里 @机器人、流式回复、长连接 WebSocket、URL 回调。发送主体：企微智能机器人/ADP 应用；消息目标：机器人会话上下文；发送模式：会话内回复、流式输出；权限模型：企业后台机器人、企业 ID、Bot ID、Secret 或扫码绑定、接入方式约束。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [已确认][官方/开发者文档镜像] 企业微信消息接口仍分为主动发送应用消息、接收消息、发送消息到群聊会话三类；主动发送应用消息是企业后台通过应用向指定成员发送单聊消息；发送到群聊会话需由企业后台创建群聊后推送到群内，且文档注明暂不支持接收群聊消息。影响：内部应用消息与客户联系/客户群仍是不同权限域。来源：[企业微信 API 概述镜像，含原文档入口](https://qiyeweixin.apifox.cn/doc-417804)
- [已确认][官方/开发者文档镜像] 企业微信客户联系“创建企业群发”接口仍强调权限和范围：自建应用只能给应用可见范围内成员推送；第三方应用需具备“客户联系-群发消息给客户和客户群”权限。影响：客户/客户群触达仍不是机器人或 Agent 任意直发；仍受客户联系权限、成员执行和频控约束。来源：[创建企业群发 API 镜像，含原文档入口](https://qiyeweixin.apifox.cn/api-10061297)
- [推断][半官方/生态] `wecom-openclaw-plugin` README 中的 proactive messaging 更可能指 Bot/Agent 双模式下对内部用户、群、部门、标签或自建应用场景的受控投递能力。理由：README 同时强调 DM/Group policy、access control、Bot/Agent mode；企业微信客户联系群发文档未出现“AI Agent 可直接向客户/客户群发送”的新说明。来源：[wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)、[创建企业群发 API 镜像](https://qiyeweixin.apifox.cn/api-10061297)
- [未确认][社区] V2EX 2026-05 讨论“企业微信外部群如何接入 AI 机器人”时，发帖者描述企微自带智能机器人只能在内部群使用，微信客服更多是 1v1 单聊，不满足外部群内直接回复需求。影响：这是社区痛点验证，不是官方限制声明；但与 GitHub issue #90 指向一致，说明外部客户群仍未被生态视为已解决。来源：[V2EX：企业微信外部群如何接入 AI 机器人](https://www.v2ex.com/t/1212320)

影响拆解：

- 发送主体：确认存在企微智能机器人、ADP 应用发布渠道、企业微信自建应用、群机器人、客户联系群发任务；未确认存在“任意 AI Agent 身份”。
- 发送目标：确认机器人单聊/群聊、内部成员、内部群、部门/标签、自建应用可见范围；客户/客户群仍需客户联系体系，不是机器人直接开放目标。
- 发送模式：长连接、URL 回调、流式回复、会话内回复继续增强；客户联系仍是群发任务/成员确认/频控模型。
- 权限模型：Bot ID/Secret、企业 ID、扫码绑定、企业后台可见范围、客户联系权限、应用可见范围仍是核心边界。
- 运营限制：未发现外部联系人、客户群、客户联系群发频控或成员确认要求被取消。
