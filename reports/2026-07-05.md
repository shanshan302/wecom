# 企业微信 AI / Agent 情报日报 - 2026-07-05

## 今日结论

暂无发现“企业微信允许 Agent 更广泛地直接向外部联系人、客户群或任意会话主动发消息”的可信新证据。

今天的可确认变化主要是生态热度延续，而非 API 权限变化：`WeComTeam/wecom-cli`、`wecom-openclaw-plugin`、企业微信智能机器人 SDK 仓库在 GitHub 的 `updated_at` 于 2026-07-04 仍有变化，但最新代码提交仍停留在 2026-07-02 或更早；公开文档层面仍显示企业微信智能机器人是单聊/群聊会话内的机器人通道，腾讯云 ADP 的企微智能机器人发布能力仍围绕长连接/Webhook 两种接入模式，没有看到发送主体、目标范围、权限模型或运营限制被放宽。

## 官方与高可信更新

- [半官方/生态][更新时间 2026-07-03] 腾讯云 ADP《将应用发布到企微智能机器人》仍是近期最重要的高可信更新：文档把接入方式拆成 `长连接 WebSocket（推荐）` 与 `URL 回调 Webhook`，场景仍是用户单聊智能机器人或群里 @ 智能机器人。影响：工程接入更清楚，但发送主体仍是企微智能机器人，不是任意 Agent 身份；发送目标仍是机器人会话上下文。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [半官方/生态][更新时间 2026-07-03] 同一 ADP 文档继续保留关键限制：一个应用只能发布到一个智能机器人；机器人只能在用户发消息后的短时窗口内回答；同一机器人同一时间只能保持一种接入方式；长连接同一时间只能保持一个有效连接。影响：没有看到多机器人路由、批量主动触达、外部联系人或客户群目标放宽。来源：[腾讯云 ADP：企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [官方][最后更新 2026-05-25] 企业微信开发者中心《智能机器人长连接》仍是判断边界的核心来源：回调上下文区分 `single` 单聊与 `group` 群聊，机器人可以在会话内发送或刷新消息，消息类型包含 markdown、卡片、文件、图片、语音、视频等。影响：这是机器人已参与会话内的发送/刷新能力，不等同于客户联系或客户群的任意主动外发。来源：[企业微信开发者中心：智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [官方][访问日期 2026-07-05] 企业微信开发者中心《创建企业群发》仍显示客户联系群发依赖 `external_userid`、`chat_id`、`msg_template` 等任务字段，本质是创建群发任务。影响：客户联系/客户群触达仍不是 Agent 直接绕过成员确认的开放发送通道。来源：[企业微信开发者中心：创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)

## 消息发送与 API 开放迹象

- [已确认][官方][最后更新 2026-05-25] 企业微信智能机器人长连接支持在机器人单聊/群聊上下文内发送或刷新消息。影响：发送目标是机器人单聊或机器人所在群聊，不是外部联系人、客户群或任意会话；这是确认能力，不应推断为客户联系 API 放宽。来源：[智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [已确认][半官方/生态][更新时间 2026-07-03] 腾讯云 ADP 企微智能机器人发布路径支持长连接和 Webhook，公开场景仍是单聊智能机器人、群里 @ 智能机器人。影响：发送模式偏回复/会话内流式输出，不是批量主动消息。来源：[ADP 企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [已确认][官方][访问日期 2026-07-05] 企业微信客户联系《创建企业群发》仍是创建群发任务，并区分客户 `external_userid` 与客户群 `chat_id`。影响：公开客户触达仍是群发任务模型，不是 Agent 直接向客户/客户群自由发送。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [已确认][官方/镜像][访问日期 2026-07-05] 新客户欢迎语和微信客服发送消息仍属于事件触发或客服会话窗口能力。影响：客户侧消息依然受触发条件、会话窗口、权限和频控约束。来源：[企业微信开发者中心](https://developer.work.weixin.qq.com/)、[Apifox 镜像：发送新客户欢迎语](https://qiyeweixin.apifox.cn/api-10061298)、[Apifox 镜像：微信客服发送消息](https://qiyeweixin.apifox.cn/api-10061328)

影响拆解：

- 发送主体：新增/高热信号仍集中在 `企微智能机器人`、`企业微信应用`、`wecom-cli` 工具链，没有出现“Agent 可作为企业主体任意发送”的证据。
- 发送目标：最清晰的是机器人单聊、机器人所在群聊、客户联系群发任务；外部联系人和客户群没有看到自由主动消息能力。
- 发送模式：长连接强化了流式回复和会话内刷新；客户联系侧仍偏成员确认、事件触发或客服窗口。
- 权限模型：`Bot ID / Secret`、回调 URL、Token、EncodingAESKey、可信 IP、企业主体配置等约束继续存在。
- 运营限制：单机器人单有效连接、一个应用一个机器人、短时回复窗口、返回长度限制等仍应按已知约束设计。
