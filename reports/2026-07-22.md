# 企业微信 AI / Agent 情报日报 - 2026-07-22

## 今日结论

**暂无“企业微信已全面放开 Agent 任意主动触达”的官方确认，但开放路径继续变清晰。** 近几日最值得注意的是腾讯云开发者 MCP 广场于 2026-07-19 收录“企业微信机器人消息服务器”；与此同时，企业微信团队维护的 OpenClaw 插件已经在 Agent 模式中公开定时任务和主动投递能力。不过二者都没有改变底层权限模型：机器人对话仍以用户发起、限时回复为主；主动消息依赖企业应用身份、明确目标、可信 IP 与既有配额。面向外部联系人/客户群的触达，仍应按客户联系群发接口与合规流程理解，不能推断为 Agent 可任意直发。

## 官方与高可信更新

- **腾讯云 ADP 4.0 / Claw 模式继续强调多渠道发布**（官方，页面近期更新；产品动态按月列示至 2026-05）：官方活动页明确 Claw 应用可发布到“企微应用/机器人”，连接器体系支持 MCP 自定义连接器；产品动态还列出企业共享 Skill 审批与治理。这增强的是 **Agent 构建、连接和分发能力**，并未宣布企业微信侧新增消息对象或放宽发送限制。[ADP 活动页](https://cloud.tencent.com/act/pro/adp-pu-activity)｜[ADP 产品动态](https://cloud.tencent.com/document/product/1759/104191)
- **ADP 发布到企微智能机器人仍是响应式链路**（官方，最后更新 2026-03-05）：支持单聊和群聊流式回复，但用户需先发消息或在群内 @ 机器人；机器人须在 3 分钟内回答，单次最多返回 2048 字节，一个应用只能发布到一个智能机器人，并需同主体/关联主体备案域名转发回调。[发布到企微智能机器人](https://intl.cloud.tencent.com/zh/document/product/1254/78022)
- **WeComTeam 两个官方仓库本月暂无新增提交**（官方开源，检查于 2026-07-22）：`wecom-cli` 最新提交仍为 2026-07-02（新增待办支持、智能表格更新）；`wecom-openclaw-plugin` 最新提交同为 2026-07-02（企业级待办和文档基础表格支持）。因此今天不把旧提交包装成新发布。[wecom-cli](https://github.com/WeComTeam/wecom-cli)｜[wecom-openclaw-plugin](https://github.com/WeComTeam/wecom-openclaw-plugin)

## AI / Agent / MCP / 智能机器人动态

- **腾讯云 MCP 广场收录企业微信机器人消息服务器**（半官方/生态，发布 2026-07-19）：页面将其描述为符合 MCP 标准的企业微信机器人服务器实现，并链接第三方 GitHub 作者项目。它说明“Agent/MCP → 企微机器人”工具化热度上升，但 **收录不等于腾讯或企微对实现、安全性、权限范围作官方背书**。[腾讯云 MCP 广场条目](https://developer.cloud.tencent.com/mcp/server/10854)
- **官方 WeCom OpenClaw 插件公开主动消息与定时投递用法**（官方开源，README 检查于 2026-07-22；仓库最新提交 2026-07-02）：README 明示支持 Bot 与 Agent 双模式、单聊/群聊、流式回复及 proactive messaging；定时任务通过 OpenClaw Cron 的 Agent outbound 通道，可投递到用户、部门、标签或群聊。关键限制是必须配置 Agent 模式、服务端 IP 进入企微可信 IP 白名单，目标也须属于应用可达范围。[WeComTeam/wecom-openclaw-plugin](https://github.com/WeComTeam/wecom-openclaw-plugin)
- **wecom-cli 继续把企微能力暴露给人类与 AI Agent**（官方开源，README 检查于 2026-07-22）：公开能力覆盖通讯录、待办、会议、消息、日程、文档与智能表格；README 仍注明当前仅对不超过 10 人企业开放使用。它是重要的 Agent 操作入口，但此开放范围提示其仍处早期/受控阶段。[WeComTeam/wecom-cli](https://github.com/WeComTeam/wecom-cli)

## 消息发送与 API 开放迹象

- **已确认：企业内部应用身份可主动发送，目标受可见范围和接口权限控制**（官方）：OpenClaw 官方企微插件把定时消息路由到 Agent outbound，目标格式覆盖 `user`、`party/dept`、`tag` 与群聊；这改变的是上层 Agent 编排方式，不是新增无边界发送者。[插件说明](https://github.com/WeComTeam/wecom-openclaw-plugin)｜[企业微信发送应用消息文档](https://developer.work.weixin.qq.com/document/path/90236)
- **已确认：智能机器人通道主要是“用户触发—机器人限时回复”**（官方，2026-03-05）：发送者是机器人，目标是与机器人单聊的成员或 @ 机器人的群；模式为回调后的流式响应，3 分钟窗口、2048 字节等运营限制仍在。[ADP 企微智能机器人渠道](https://intl.cloud.tencent.com/zh/document/product/1254/78022)
- **已确认：外部联系人与客户群触达仍是专门的客户联系群发机制**（官方）：应继续区分企业内部应用消息、群机器人 webhook、智能机器人回复、客户联系群发四类通道；客户/客户群发送涉及成员授权、客户关系、任务执行与频率等约束，不能用内部 Agent 主动消息能力类推。[企业微信客户联系群发文档](https://developer.work.weixin.qq.com/document/path/92135)
- **推断：更广泛 Agent 主动发送更可能先通过“官方插件/CLI + 企业应用权限”渐进落地**（半官方/生态）：现有信号集中在工具封装、目标路由和定时编排，没有看到企业微信官方新增“Agent 身份”、放开外部客户任意主动发送，或取消可信 IP/可见范围/频控的公告。

## 未证实或低可信信号

- **“MCP Server 可以向企微发消息”不等于“企微开放 Agent 任意发送”**（社区，2026-07-19）：MCP 广场项目是第三方实现，具体 sender、target、凭证存储、频率限制与合规边界需审代码和企微接口调用后再判断。
- **“OpenClaw 插件支持 proactive messaging”不等于机器人 webhook 获得主动推送权**（社区常见误读）：官方插件 README 明确定时任务走 Agent outbound，并要求 Agent 模式与可信 IP；应避免把企业应用发送能力误标成智能机器人权限扩张。
