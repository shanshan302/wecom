# 企业微信 AI / Agent 情报日报 - 2026-07-23

## 今日结论

**暂无重大变化：未发现企业微信新增“Agent 可面向任意成员、外部联系人或客户群主动发送”的官方能力。** 近期官方更新的主线仍是腾讯云 ADP/Claw 的构建、连接、分发和治理，以及把企业微信智能机器人作为远程对话入口；这不等于企业微信放宽底层发送者身份、目标范围、触发方式、可信 IP、可见范围或频率限制。内部主动触达仍应使用企业应用消息能力，外部联系人及客户群仍应使用客户联系专用接口与合规流程。

## 官方与高可信更新

- **ADP 4.0 升级为企业级 AgentOps 平台**（官方，发布 2026-07-17）：腾讯云 ADP 团队明确新增 Claw 模式、云端 Agent Harness、连接器、Skills、MCP、权限治理与运行观测，重点是 Agent 构建、运行和治理；公告没有宣布企业微信侧新增消息对象或放松主动发送权限。[腾讯云 ADP 官方博客](https://adp.tencent.com/zh/blog/adp-version-4-agentops-platform)｜[腾讯官方大会回顾（2026-06-05）](https://www.tencent.com/zh-cn/tencent-cloud-debuts-productivity-agent-suite-creating-a-new-gateway-to-ai-for-users-and-enterprises/)
- **ADP 智能工作台继续支持企微智能机器人作为远程入口**（官方，活动期 2026-07-06 至 2026-07-31）：官方活动页明确可配置微信 ClawBot、企业微信智能机器人作为远程对话入口，并同步渠道历史会话。这里改变的是 **Agent 使用入口和会话连续性**，不是新增主动群发通道。[智能体平台套餐活动页](https://cloud.tencent.com.cn/act/pro/openclaw-in-adp)
- **WeComTeam 官方仓库今日无新提交**（官方开源，检查于 2026-07-23）：`wecom-cli` 与 `wecom-openclaw-plugin` 最新提交仍停留在 2026-07-02；前者最近增加待办与智能表格编辑，后者增加企业级待办与文档基础表格支持。旧提交不作为今日新品发布。[wecom-cli](https://github.com/WeComTeam/wecom-cli)｜[wecom-openclaw-plugin](https://github.com/WeComTeam/wecom-openclaw-plugin)

## AI / Agent / MCP / 智能机器人动态

- **Lighthouse Agent 把微信/企微纳入统一“消息通道”管理**（官方，更新 2026-07-16）：腾讯轻量云称其可连接和管理微信、企业微信等通道，并提供多 Agent、技能、定时任务与观测能力。它是云上 Agent 管理层的增强；页面未给出企微发送权限扩张的接口级证据。[腾讯云开发者社区官方账号](https://developer.cloud.tencent.com/article/2710300)
- **ADP Claw Mode 已提供 API 全生命周期示例**（官方，更新 2026-07-15）：文档演示从创建到发布 Claw 应用的 API 流程，说明 Agent 自动化部署接口趋于完整；但“发布 Agent”与“获得企微任意消息发送权限”是两件事。[从零搭建 Claw 模式应用](https://cloud.tencent.cn/document/product/1759/133869)
- **官方企微 OpenClaw 插件仍是最直接的 Agent 接入证据**（官方开源，README 检查于 2026-07-23）：支持 Bot/Agent 双模式、单聊/群聊、流式回复及 Agent 模式下的主动/定时投递。主动能力依赖企业应用身份、可信 IP 和目标可达范围，不能解读为群机器人 webhook 主动推送权。[WeComTeam/wecom-openclaw-plugin](https://github.com/WeComTeam/wecom-openclaw-plugin)

## 消息发送与 API 开放迹象

- **已确认——内部企业应用可主动发送**（官方）：发送者是企业自建应用，目标可为可见范围内成员、部门或标签等，受应用权限、可见范围、可信 IP 和接口限制约束；Agent/CLI 只是上层编排器。[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)
- **已确认——企微智能机器人以用户触发后的回复为主**（官方，文档更新 2026-03-05）：ADP 渠道要求用户先私聊或在群内 @ 机器人，机器人在规定窗口内回复；这与企业应用主动消息不是同一发送模式。[ADP 发布到企微智能机器人](https://intl.cloud.tencent.com/zh/document/product/1254/78022)
- **已确认——外部联系人/客户群触达仍走客户联系群发机制**（官方）：发送者、客户关系、成员授权、任务执行、目标与频控均有专门约束，不能由内部应用消息或 Agent 主动投递能力类推。[客户联系群发文档](https://developer.work.weixin.qq.com/document/path/92135)
- **推断——更广泛的 Agent 发送会先沿“受治理的企业应用 + 官方连接器”演进**（半官方/生态）：ADP AgentOps、Lighthouse 通道管理和官方 OpenClaw 插件都在强化治理与编排，但截至今日没有证据显示企业微信新增独立 Agent 身份、扩大外部目标、取消用户触发，或放松运营限制。
