# 企业微信 AI / Agent 情报日报 - 2026-06-24

## 今日结论

暂无“企业微信向更广泛 API / Agent 直接主动发消息”方向的重大新增官方变化。

今天复核后，仍然成立的高可信判断只有三条：

- 企业微信已经明确支持 `API 模式（长连接）智能机器人` 对接 OpenClaw，并确认该模式支持“被动回复多条消息、主动向用户发消息”；但这是机器人受控通道，不等于开放任意成员身份代发。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（访问日期：2026-06-24，置信度：官方）
- 企业微信当前面对客户 / 客户群的官方触达能力，仍是“创建群发任务并通知成员发送”，不是 API 直接代表企业向客户群即时出站。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（访问日期：2026-06-24，置信度：官方）
- 腾讯云 ADP 生态仍在增强“发布到企业微信应用 / 发布到企微智能机器人 / MCP 连接器”这三条路径，但今天没有看到它带来企业微信消息权限边界的官方放宽。来源：[将应用发布到企业微信应用](https://cloud.tencent.com/document/product/1759/119175)、[将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)、[新建连接器](https://cloud.tencent.com/document/product/1759/115873)（访问日期：2026-06-24，置信度：半官方/生态）

## 官方与高可信更新

- [官方][访问日期 2026-06-24] 企业微信帮助中心 `OpenClaw 接入企业微信智能机器人` 仍明确写着：通过 `API模式（长连接）` 创建的智能机器人，支持“被动回复多条消息、主动向用户发消息”。这说明“主动消息”已确认存在，但主体是机器人，不是任意员工身份。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [官方][访问日期 2026-06-24] 同一帮助页继续明确：`目前配置 OpenClaw 接入企业微信智能机器人，暂不支持在外部群 / 客户群中使用`。这直接否定了“官方机器人能力已经扩展到客户群运营”的推断。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [官方][访问日期 2026-06-24] 同一帮助页继续写明：`3月13日更新的 5.0.7 版本已支持机器人通过 MCP 创建文档、智能表格`，并保留“对话授权”“手动授权（streamableHTTP URL / JSON Config）”两种安装方式。今天看到的是存量能力仍然在线，不是今日新增。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [官方][访问日期 2026-06-24] `如何获取 Bot ID 和 Secret` 页面仍说明：API 模式、长连接方式下会自动生成并展示 `Bot ID` 与 `Secret`。这继续表明当前 OpenClaw / Agent 集成仍围绕专门机器人凭证展开。来源：[如何获取 Bot ID 和 Secret](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21677)
- [官方][访问日期 2026-06-24] `发送应用消息` 文档的投递对象仍是 `touser` / `toparty` / `totag`。未看到外部联系人、客户群或“任意会话主动直发”的官方新增目标。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)
- [官方][访问日期 2026-06-24] `创建企业群发` 文档仍明确：`不会直接发送消息给客户/客户群，需要成员确认后才会执行发送`，且 `仅会推送给最后跟客户进行聊天互动的企业成员`。这说明客户触达依然绑定成员关系、成员确认与成员执行。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [官方][访问日期 2026-06-24] `创建企业群发` 文档当前口径仍是：每位客户 / 每个客户群每月最多可接收条数为`当月天数`。这继续说明旧“4 条 / 月”口径不是现行官方标准。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [官方][GitHub API 访问日期 2026-06-24] `WecomTeam/wecom-cli` 仓库仍将自己定义为“让人类和 AI Agent 都能在终端中操作企业微信”，GitHub API 显示 `pushed_at = 2026-05-27T09:47:36Z`。今天没有看到 6 月下旬的新提交把消息权限边界再往外推。来源：[WecomTeam/wecom-cli](https://github.com/WecomTeam/wecom-cli)
- [官方][GitHub API 访问日期 2026-06-24] `WecomTeam/wecom-openclaw-plugin` 仓库说明仍强调“无缝对接企业微信工作流，集成文档、智能表格、日历能力”，GitHub API 显示 `pushed_at = 2026-05-15T04:29:38Z`。今天没有看到新增 README 或提交宣告客户群 / 外部联系人发送边界变化。来源：[WecomTeam/wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)

## AI / Agent / MCP / 智能机器人动态

- [官方][访问日期 2026-06-24] 企业微信官方帮助页仍把 `MCP 创建文档、智能表格` 放在 OpenClaw 机器人接入主路径中，而且支持对话授权与手动安装两种配置。这表明企业微信对 Agent 的官方叙事，已经稳定聚焦在“机器人 + MCP 工具调用 + 原生办公资产”组合。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [半官方/生态][访问日期 2026-06-24] 腾讯云 ADP `新建连接器` 文档仍明确：当预置连接器不满足需求时，可通过 `MCP（Model Context Protocol）` 接入自定义连接器。这个信号支持“ADP -> 内部系统 / 第三方 SaaS / 企微工具链”的工程化扩展判断，但不是企业微信自身接口放宽。来源：[新建连接器](https://cloud.tencent.com/document/product/1759/115873)
- [半官方/生态][访问日期 2026-06-24] 腾讯云 ADP 文档仍同时保留 `将应用发布到企业微信应用` 与 `将应用发布到企微智能机器人` 两条路径，说明腾讯云仍在同时押注“工作台入口”和“会话机器人入口”。这属于生态集成加强，不等于 WeCom 开放更宽的消息目标面。来源：[将应用发布到企业微信应用](https://cloud.tencent.com/document/product/1759/119175)、[将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [半官方/生态][访问日期 2026-06-24] 腾讯云 ADP 产品页当前醒目卡片出现 `ADP 上线智能工作台`，说明腾讯云近期的产品动态重心之一是员工侧智能助手工作台，而不是专门针对企业微信消息开放的新公告。来源：[腾讯云智能体开发平台 ADP](https://cloud.tencent.com/product/adp)
- [官方][GitHub API 访问日期 2026-06-24] `openclaw/openclaw` 仓库 `pushed_at = 2026-06-24T01:01:44Z`，说明 OpenClaw 主项目仍在快速演进；但今天没有看到来自企业微信官方渠道的同步声明，把这些变化映射成新的 WeCom 发送权限或新目标面。来源：[openclaw/openclaw](https://github.com/openclaw/openclaw)

## 消息发送与 API 开放迹象

- [官方][访问日期 2026-06-24] 已确认：`发送应用消息` 仍是企业内投递模型，目标对象仍为 `用户 / 部门 / 标签`。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)
- [官方][访问日期 2026-06-24] 已确认：`创建企业群发` 仍是“建任务 -> 通知成员 -> 成员确认 -> 发送”，不是 API 直接代企业向客户 / 客户群即时发送。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)
- [官方][访问日期 2026-06-24] 已确认：企业微信帮助中心继续明确 OpenClaw 机器人“主动向用户发消息”，但同页同时写明“暂不支持在外部群 / 客户群中使用”。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)
- [官方][访问日期 2026-06-24] 已确认：OpenClaw / 长连接能力的权限模型仍由 `Bot ID / Secret`、机器人可见范围、长连接权限发放等控制。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)、[如何获取 Bot ID 和 Secret](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21677)
- [推断][基于上述官方页面综合判断] 今日没有发现任何可信新增信号表明：企业微信已经允许更广泛的 API 或 Agent 直接主动向外部联系人、客户群、或任意员工会话发送消息。当前最接近“主动出站”的仍是三条受控通道：`智能机器人 API 模式`、`自建应用消息`、`客户联系群发任务`。

影响拆解：

- 发送主体：仍是 `机器人`、`应用`、`成员确认后的群发任务`，未见普通成员身份 API 化代发。
- 发送目标：内部消息仍限 `用户 / 部门 / 标签`；客户侧仍限 `客户 / 客户群群发任务`；OpenClaw 官方仍不进 `外部群 / 客户群`。
- 发送模式：机器人通道支持被动回复 + 主动消息；客户联系仍是任务式 / 确认式；未见“任意实时外部会话直发”放开。
- 权限模型：继续依赖 `Bot ID / Secret`、应用 `Secret`、成员关系归属、可见范围、管理员开通长连接权限等机制。
- 运营限制：当前官方群发上限仍是“当月天数”，旧“4 条 / 月”说法仍应视为过时二手口径。
