# 企业微信 AI / Agent 情报日报 - 2026-07-06

## 今日结论

暂无发现“企业微信允许 Agent 更广泛地直接向外部联系人、客户群或任意会话主动发消息”的官方新证据。

今天值得注意的变化不是企业微信客户联系 API 放宽，而是生态表述更明确：`WeComTeam/wecom-openclaw-plugin` README 明确写到支持 Bot mode、Agent mode、DM、群聊、流式回复和 `proactive messaging`。但结合腾讯云 ADP 企微智能机器人文档、企业微信智能机器人长连接文档、客户联系群发接口说明来看，这里的主动消息仍应理解为机器人/自建应用/插件配置范围内的出站能力，不等同于 Agent 可绕过成员确认、客户联系权限、外部群限制或运营频控直接触达外部联系人和客户群。

## 官方与高可信更新

- [半官方/生态][更新时间 2026-07-03] 腾讯云 ADP《将应用发布到企微智能机器人》仍是近期最重要的高可信文档。文档显示企微智能机器人发布渠道支持长连接模式（扫码绑定、填写表单绑定）与 URL 回调模式；长连接需要企业 ID、Bot ID、Secret，URL 回调模式需要备案且主体匹配或有关联关系的域名。影响：接入路径更清楚，但发送主体仍是“企微智能机器人/ADP 应用发布渠道”，不是任意 Agent 身份；目标仍围绕机器人会话和发布渠道。来源：[腾讯云 ADP：将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [官方][最后更新 2026-05-25，检索日期 2026-07-06] 企业微信开发者中心《智能机器人长连接》仍是机器人能力边界的核心来源。公开页面仍指向智能机器人通过 WebSocket 长连接或回调方式接入，处理单聊/群聊上下文中的消息收发。影响：确认机器人会话内的发送/刷新/回复能力，但没有看到外部联系人、客户群、客户联系接口被并入长连接机器人通道。来源：[企业微信开发者中心：智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [官方/镜像][访问日期 2026-07-06] 企业微信客户联系《创建企业群发》接口仍是“添加企业群发消息任务并通知成员发送给相关客户或客户群”，而不是直接代成员完成发送；镜像文档还保留第三方应用需具备“客户联系-群发消息给客户和客户群”权限、同一企业每自然月对同一客户/客户群发送上限等限制。影响：客户侧触达仍是任务/成员确认/频控模型。来源：[企业微信开发者中心：创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)、[Apifox 镜像：创建企业群发](https://qiyeweixin.apifox.cn/api-10061297)、[Hexdocs 镜像：WeChat.Work.Customer.GroupMsg](https://hexdocs.pm/wechat_sdk/WeChat.Work.Customer.GroupMsg.html)
- [半官方/生态][产品动态 2026-05 至 2026-03] 腾讯云 ADP 产品动态显示：2026-05 插件广场升级为“连接器与工具”，连接器可在授权或必要配置后访问外部系统并完成查询、创建、更新、通知等操作；2026-03 ClawPro 支持微信客服对接渠道、企业微信智能机器人渠道扫码授权方式对接。影响：腾讯云 Agent 平台在扩大连接器和渠道分发，但不是企业微信客户联系 API 的新开放。来源：[腾讯云 ADP 产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [半官方/生态][访问日期 2026-07-06] 腾讯云 ADP 产品页案例显示，腾讯学堂 AI 问答助手上线到企业微信智能机器人、微信小程序等入口；雍禾植发案例提到 ADP 应用与企业微信对接，辅助销售团队获取业务知识与传递客户需求。影响：企业微信作为 Agent/AI 应用入口继续被官方产品化呈现，但案例没有披露新的消息发送权限。来源：[腾讯云智能体开发平台产品页](https://cloud.tencent.com/product/adp)

## 消息发送与 API 开放迹象

- [已确认][官方/生态] 企微智能机器人长连接/ADP 发布渠道支持机器人会话内消息收发，并可通过 Bot ID/Secret 或扫码授权接入。影响：发送主体是智能机器人；发送目标是机器人可见范围内的用户/群或机器人上下文；发送模式是会话内回复、流式回复或受配置限制的出站消息；权限模型依赖 Bot ID/Secret、企业 ID、可见范围、长连接或回调配置。来源：[ADP 企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)、[智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463)
- [已确认][官方/镜像] 客户联系“创建企业群发”仍是创建群发任务并通知成员发送，第三方应用需具备客户联系群发权限。影响：目标可以是客户或客户群，但发送模式不是 Agent 直接发出；成员确认、月度触达上限、应用权限仍是关键限制。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)、[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061297)
- [已确认][社区/产品实践] 内部群聊 Webhook 群机器人仍可用于自动通知、Markdown、文件等推送，但社区资料普遍提示外部群/客户群不等同支持。影响：发送主体是群机器人，目标通常是内部群，不能推断到外部客户群。来源：[明道云：如何向企业微信群发送消息](https://help.mingdao.com/wecom/group-robot/)、[企业微信群机器人文档入口](https://developer.work.weixin.qq.com/document/path/91770)
- [推断][官方/生态] `wecom-openclaw-plugin` 的 proactive messaging 更可能覆盖自建应用 Agent mode、内部用户/部门/标签、Bot/Agent fallback 等企业内部协作对象，而不是客户联系 API 的自由外发。理由：README 同时强调 access control、DM/group policy、Bot/Agent config；官方客户联系群发文档仍保留任务/成员确认/权限模型。来源：[wecom-openclaw-plugin README](https://github.com/WeComTeam/wecom-openclaw-plugin/blob/main/README.md)、[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)

影响拆解：

- 发送主体：明确存在 `企微智能机器人`、`企业微信自建应用/Agent mode`、`企业微信群机器人`、`客户联系群发任务`，但没有看到“任意 Agent 身份”作为新主体。
- 发送目标：机器人单聊/群聊、内部用户/部门/标签、内部群、客户/客户群群发任务分别属于不同通道；今天没有看到外部联系人或客户群自由主动消息能力。
- 发送模式：长连接与 OpenClaw 插件强化主动/流式/回退投递；客户联系仍是任务通知成员发送。
- 权限模型：Bot ID/Secret、Token/EncodingAESKey、企业 ID、备案域名、可见范围、客户联系权限、DM/group allowlist 继续存在。
- 运营限制：客户联系群发仍受成员确认和频控；机器人/插件的 proactive messaging 应按机器人/应用配置与企业微信官方限制设计。
