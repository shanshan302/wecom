# 企业微信 AI / Agent 情报日报 - 2026-07-17

## 今日结论

暂无重大变化：截至 2026-07-17，未发现企业微信官方宣布扩大 AI Agent 对微信客户、外部联系人或客户群的任意主动消息权限。

今天最值得关注的是一次能力边界纠偏。企业微信团队维护的 `wecom-openclaw-plugin` README 将定时任务目标 `group:<id>` 标注为 “External group chat”，并举出 `wr...` 示例；但当前源码把这类 `chatid` 送往 `/cgi-bin/appchat/send`（应用群聊），而企业微信客户/客户群的官方触达仍走 `/cgi-bin/externalcontact/add_msg_template`，且调用后仍需成员确认。由此不能确认插件已支持向客户群直接主动发送；更可能是 README 标签过宽、目标 ID 示例或实现存在待澄清的不一致。

`WeComTeam/wecom-cli` 与 `WeComTeam/wecom-openclaw-plugin` 的最新提交仍分别停留在 2026-07-02；`wecom-cli` issue #90“只能给同事、不能给客户发消息？”仍为 open、0 条回复。明日深圳 Agent Go 活动仍开放申请，但未公布企业微信专项议程或明确讲者。

- [官方][接口概述镜像最近修改 2026-05-27；检索 2026-07-17] 企业微信消息接口仍区分应用单聊、回调回复、应用群聊；应用群聊由企业后台先创建，再调用接口推送，属于企业内部应用消息域。来源：[企业微信消息接口概述镜像（含官方文档入口）](https://s.apifox.cn/apidoc/docs-site/406014/doc-417895)

## 消息发送与 API 开放迹象

- [已确认][半官方/生态] 插件 `target.ts` 将 `group:`、`chat:`、`wr...`、`wc...` 统一解析成 `chatid`；`api-client.ts` 在存在 `chatid` 时调用 `SEND_APPCHAT`，即 `/cgi-bin/appchat/send`。这确认了当前代码路径，但不确认目标一定能是客户群。来源：[target.ts](https://github.com/WecomTeam/wecom-openclaw-plugin/blob/main/src/target.ts)、[api-client.ts](https://github.com/WecomTeam/wecom-openclaw-plugin/blob/main/src/agent/api-client.ts)
- [已确认][官方/文档镜像] `/cgi-bin/appchat/send` 是“应用推送消息”接口，受每企业每分钟/小时人次和成员日接收量等限制；企业微信消息概述把它描述为企业后台创建群聊后由应用推送，不属于客户联系群发。来源：[应用推送消息](https://s.apifox.cn/apidoc/docs-site/406014/api-10061357)、[消息接口概述](https://s.apifox.cn/apidoc/docs-site/406014/doc-417895)
- [已确认][官方/文档镜像] 客户/客户群触达仍使用 `/cgi-bin/externalcontact/add_msg_template` 创建群发任务；调用不会直接发送，需成员确认后执行，同一企业每自然月对一个客户/客户群最多发送 4 条企业群发。来源：[创建企业群发](https://apifox.com/apidoc/docs-site/406014/api-10061297)
- [未确认][半官方/生态] README 把 `group:wr...` 写成 “External group chat”，但源码未调用客户联系群发接口，且插件没有给出成员确认、客户群频控或 `externalcontact` 权限处理。当前应视为文档/实现待澄清，而不是“Agent 可直接向客户群主动发消息”。来源：[插件 README](https://github.com/WecomTeam/wecom-openclaw-plugin#cronjob-scheduled-tasks)

影响拆解：

- 发送主体：确认是企微智能机器人或企业自建应用；未出现脱离企业授权的通用 Agent 身份。
- 消息目标：确认内部用户、部门、标签和应用群聊；README 声称的外部群目标尚未由对应官方客户联系调用链证实。
- 发送模式：会话回复、流式回复、Cron/主动投递与 Bot → Agent 回退已具备；客户群仍是创建群发任务后成员确认。
- 权限模型：Bot ID/Secret、自建应用 CorpID/CorpSecret/AgentId、应用可见范围、可信 IP，以及客户联系专用权限仍分别生效。
- 运营限制：应用群聊与客户群发各有独立限额；未发现客户群每月触达上限、成员确认或合规边界被取消。
