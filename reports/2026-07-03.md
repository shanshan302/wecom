# 企业微信 AI / Agent 情报日报 - 2026-07-03

## 今日结论

暂无重大能力边界变化。

今天查到的高信号新增，主要是 `2026-07-02` 企业微信团队两个官方 GitHub 仓库继续推进 Agent 工具链，而不是企业微信消息权限模型放宽：

- [官方][更新时间 2026-07-02] `WecomTeam/wecom-cli` 连续合入 `Todo` 和 `WeCom Sheet` 编辑相关提交，说明企业微信团队仍在加速把 `待办`、`文档/智能表格`、`消息` 等能力封装成面向人类和 AI Agent 的 CLI 能力，但这不等于开放新的消息发送边界。来源：[仓库](https://github.com/WecomTeam/wecom-cli)、[最新提交 1](https://github.com/WecomTeam/wecom-cli/commit/9d2aeaf37fe6014801c853498afe20d06410feda)、[最新提交 2](https://github.com/WecomTeam/wecom-cli/commit/bae1cc3e147716c23936b9d15a5611f073578f2d)
- [官方][更新时间 2026-07-02] `WecomTeam/wecom-openclaw-plugin` 新提交 `Enterprise-grade todo support and basic table support in documents`，方向同样是增强 Agent 在企业微信工作流内处理 `待办` 和 `表格/文档` 的执行能力。来源：[仓库](https://github.com/WecomTeam/wecom-openclaw-plugin)、[最新提交](https://github.com/WecomTeam/wecom-openclaw-plugin/commit/1a91ef7300de7274de8d74e4a566cf3b6e569a25)
- [半官方/生态][最近更新时间 2026-06-11] 腾讯云 ADP 最近一轮公开产品动态仍把企业微信放在 `智能机器人分发`、`远程终端渠道`、`定时任务结果推送` 这条线上，没有公开新增“更广泛 Agent 可直接给外部联系人/客户群主动发消息”的表述。来源：[腾讯云智能体开发平台产品动态](https://cloud.tencent.com/document/product/1759/104191)

结论上，今天更像是“Agent 工具链持续加厚”，不是“消息 API 显著放开”。

## 消息发送与 API 开放迹象

- [已确认][半官方/生态][最近更新时间 2026-03-05] 腾讯云官方文档明确，`企微智能机器人` 与 `企业微信应用` 是两条不同接入路径。前者强调 `流式回复`、`群里 @机器人 / 单聊机器人`；后者强调应用回调、可信 IP、AgentId / Secret。这是当前最清晰的“发送主体差异”。来源：[企微智能机器人文档](https://cloud.tencent.com/document/product/1759/121473)、[企业微信应用文档](https://cloud.tencent.com/document/product/1759/119175)
- [已确认][半官方/生态][最近更新时间 2026-03-05] 机器人路径的已公开运营限制至少包括：`一个应用只能发布到一个智能机器人`、`只能在 3 分钟内回答`。这影响的是 `发送模式` 和 `执行时限`，不是目标范围放开。来源：[企微智能机器人文档](https://cloud.tencent.com/document/product/1759/121473)
- [已确认][半官方/生态][访问日期 2026-07-03] 企业微信 API 公开镜像 `创建企业群发` 摘要仍显示：接口是 `创建任务并通知成员发送`，`不会直接发送消息给客户/客户群，需要成员确认后才会执行发送`。这意味着客户联系侧公开证据仍偏“成员确认式触达”，不是 Agent 统一直接出站。来源：[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061297)
- [已确认][半官方/生态][访问日期 2026-07-03] `发送新客户欢迎语` 镜像摘要仍显示：依赖 `welcome_code`，需在 `20 秒内` 调用且 `只可调用一次`。这属于事件触发型定向消息，不是常驻主动发送权限。来源：[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061298)
- [已确认][半官方/生态][访问日期 2026-07-03] `发送消息`（微信客服）镜像摘要仍显示：仅在用户主动发消息后的 `48 小时` 内最多可下发 `5 条消息`。这说明面向外部客户的对话发送仍被严格限制在客服场景和时间窗内。来源：[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061328)
- [已确认][半官方/生态][访问日期 2026-07-03] `应用推送消息` 镜像摘要继续显示内部群聊也有明确频控：`每企业不超过 2 万人次/分、30 万人次/小时`，且单成员在群中 `200 条/分、1 万条/天` 超过会被丢弃。来源：[Apifox 镜像](https://qiyeweixin.apifox.cn/api-10061357)
- [推断][基于今日全部高可信材料] 截至 `2026-07-03`，没有看到可信新增证据表明企业微信已允许更广泛的 Agent 直接向 `外部联系人`、`客户群`、`外部群` 做常规主动消息发送。

影响拆解：

- 发送主体：仍以 `智能机器人`、`企业微信应用`、`客户联系任务`、`微信客服场景` 为主。
- 发送目标：内部成员和机器人所在会话最清晰；客户侧仍多为 `欢迎语`、`客服时窗`、`群发任务` 这类受限模式。
- 发送模式：增强点集中在 `流式回复`、`远程终端`、`定时任务结果推送`，不是普遍开放任意会话主动出站。
- 权限模型：继续依赖 `Bot ID / Secret`、`AgentId / Secret`、`企业 ID`、`可信 IP`、`扫码授权`、`事件 code` 等多层限制。
- 运营限制：公开可见的 `3 分钟回复窗`、`48 小时客服窗口`、`群发需成员确认`、`内部群聊频控` 仍然存在。
