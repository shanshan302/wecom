# 企业微信 AI / Agent 情报日报 - 2026-07-14

## 今日结论

暂无重大变化：截至 2026-07-14，未发现企业微信官方宣布扩大 AI Agent 对外部联系人、客户群或任意会话的主动消息权限。

今天最值得关注的是腾讯云 ADP 已公开一条“无人值守定时任务 → 已绑定企业微信机器人指定会话”的主动推送路径。该能力确实改变了**发送模式**（由用户提问后回复扩展到定时主动推送），但没有改变**发送主体**（ADP 智能工作台/企微机器人）、**消息目标**（已绑定机器人的一个指定私聊或群聊会话）、**权限模型**（先绑定渠道并选定会话）或客户联系体系的运营限制。ADP 新版 API 总览同时明确开发者管理接口与终端用户对话接口权限隔离，不能据此推断 Agent 获得企业微信客户消息权限。

`WeComTeam/wecom-cli` 与 `WeComTeam/wecom-openclaw-plugin` 自 2026-07-10 日报观察点后均无新提交，也没有维护者公开确认“可直接给客户/客户群发消息”。

## 官方与高可信更新

- [官方][文档更新 2026-07-09；本次新观察，非 7 月 14 日新发布] 腾讯云 ADP 新版 API《概览》公开 API 版本 `2026-05-20`，明确开发者使用 `SecretId/SecretKey` 调用创建、配置、发布等管理接口，终端用户使用 `AppKey + UserId` 进入对话；两者权限和操作范围完全隔离。影响：ADP 的 Agent 生命周期与对话接入更标准化，但这是 ADP 自身 API，不是企业微信消息 API 扩权。来源：[腾讯云 ADP API 概览](https://cloud.tencent.com/document/product/1759/133868)
- [官方][文档更新 2026-06-11；检索 2026-07-14] ADP 智能工作台定时任务可在无人值守情况下生成报告、巡检或提醒，并把结果推送到已绑定的微信或企业微信机器人指定会话；最小执行间隔 1 小时。影响：确认 Agent 可定时主动推送，但每项任务只能选一个指定会话，微信/企微端收到该结果后不能基于该条消息继续对话。来源：[腾讯云 ADP：定时任务](https://cloud.tencent.com/document/product/1759/132761)
- [官方][最近更新 2026-06-15；检索 2026-07-14] ADP 产品概述继续确认平台支持 Claw、标准、多智能体与 Workflow，内置 200+ 连接器/工具、支持 MCP，并可发布至企业微信等渠道。影响：企业微信是 Agent 发布入口，但“发布渠道”不等于获得客户联系直发权限。来源：[腾讯云 ADP 产品概述](https://cloud.tencent.com/document/product/1759/104193/)
- [官方][版本日期 2026-06-23；检索 2026-07-14] 企业微信 5.0.9 的公开产品页继续强调智能搜索、智能总结、智能机器人和面向微信客户的服务总结；未见该版本说明开放 Agent 代成员主动给客户或客户群发消息。来源：[企业微信产品页/版本信息](https://wwurl.rtxapp.com/)

## 消息发送与 API 开放迹象

- [已确认][官方] **发送模式有增量**：ADP 定时任务可无人值守主动向已绑定的企业微信机器人会话推送结果；远程终端创建的任务会自动回推创建任务的终端会话。来源：[腾讯云 ADP：定时任务](https://cloud.tencent.com/document/product/1759/132761)
- [已确认][官方] **目标和权限未放宽**：每个定时任务只能推送到一个指定会话；机器人若存在于多个私聊/群聊，需明确选择；未绑定渠道时不可选择推送。接收结果后也不支持从该消息继续对话。来源：[腾讯云 ADP：定时任务](https://cloud.tencent.com/document/product/1759/132761)
- [已确认][官方/文档镜像含官方入口] 企业微信客户联系仍通过“创建企业群发”等专用接口与权限域运行；自建应用受可见范围限制，第三方应用需“客户联系-群发消息给客户和客户群”权限。没有证据显示智能机器人或 ADP 绕过成员执行、频控和客户联系授权。来源：[创建企业群发 API 镜像](https://qiyeweixin.apifox.cn/api-10061297)
- [推断][高可信] ADP 的 `CreateApp`、`ModifyAgent`、`CreateRelease`、对话 SSE 等接口仅管理/调用 ADP 应用。由于官方明确区分开发者和使用者鉴权，且定时推送仍要求绑定具体企微会话，因此不能把 ADP API 版本 `2026-05-20` 推断为企业微信任意主动发信 API。来源：[腾讯云 ADP API 概览](https://cloud.tencent.com/document/product/1759/133868)

影响拆解：

- 发送主体：ADP 智能工作台/已绑定企微智能机器人；未出现通用“Agent 身份”。
- 消息目标：一个已绑定的机器人私聊或群聊会话；未扩展至任意外部联系人或客户群。
- 发送模式：新增/确认定时、无人值守主动推送；会话内回复、流式回复仍继续存在。
- 权限模型：ADP 开发者密钥、应用 AppKey/UserId、企微机器人渠道绑定和指定会话共同约束。
- 运营限制：定时任务最小间隔 1 小时；一个任务一个会话；未发现客户联系频控或成员确认要求取消。
