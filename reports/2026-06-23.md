# 企业微信 AI / Agent 情报日报 - 2026-06-23

## 今日结论

暂无“更广泛开放企业微信 API / Agent 主动发消息”的重大变化。

截至 2026-06-23（Asia/Shanghai）可验证的最强官方信号仍是两条：

- 企业微信已经明确支持 `API 模式（长连接）智能机器人` 对接 OpenClaw，并确认该模式支持“被动回复多条消息”和“主动向用户发消息”；但这仍是机器人受控通道，不等于开放“任意成员身份代发”。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（发布时间可从页内时间戳换算为 2026-05-20，置信度：官方）
- 企业微信当前对“客户 / 客户群”的官方可编排发送能力，仍然是“创建群发任务并通知成员发送”，不是 API 直接代企业向客户群即时发出消息。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（访问日期 2026-06-23，置信度：官方）

今天最值得记录的变化不是新能力，而是口径纠偏：

- 许多社区/镜像页面仍在复述“同一客户/客户群每月最多 4 条”，但当前官方 `创建企业群发` 文档已写成“每位客户 / 每个客户群每月最多可接收条数为当月天数”。这意味着旧“4 条/月”说法不能再当现行规则使用。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（访问日期 2026-06-23，置信度：官方）

## 官方与高可信更新

- [官方][2026-05-20] 企业微信帮助中心新增并公开 `OpenClaw 接入企业微信智能机器人` 页面，明确写出：API 模式长连接机器人支持“被动回复多条消息、主动向用户发消息”，且 Bot ID / Secret 为该接入方式的核心凭证。该页同时说明长连接模式“无需域名 / IP 即可接收消息并返回结果”。这说明企业微信已经把 Agent 接入收敛到“智能机器人 API 模式”这个明确入口。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（置信度：官方）
- [官方][2026-05-20] 同一帮助页写明：`3月13日更新的 5.0.7 版本已支持机器人通过 MCP 创建文档、智能表格`，并提供“对话授权”和“手动授权”两种配置路径。这个信号重要，因为它不是社区二创，而是企业微信官方帮助页直接把 `MCP`、`文档`、`智能表格` 放进机器人能力叙事。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（置信度：官方）
- [官方][2026-05-20] 企业微信官方同时写明：`OpenClaw 机器人暂不支持在外部群 / 客户群中使用`。这直接卡住了“把当前官方机器人能力外延到客户群运营”这一推断。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（置信度：官方）
- [官方][访问日期 2026-06-23] `发送应用消息` 文档的发送对象仍是 `touser` / `toparty` / `totag`。这说明标准应用消息接口的收件目标仍然是企业内用户、部门、标签，而不是外部联系人或客户群。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)（置信度：官方）
- [官方][访问日期 2026-06-23] `创建企业群发` 文档继续明确两条约束：`不会直接发送消息给客户/客户群，需要成员确认后才会执行发送`，且 `仅会推送给最后跟客户进行聊天互动的企业成员`。这两条共同表明：面向客户侧的触达仍然是“成员执行 / 成员确认 / 成员关系归属”的权限模型。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（置信度：官方）
- [官方][访问日期 2026-06-23] WeComTeam 官方仓库 `wecom-cli` README 仍将自己定义为“让人类和 AI Agent 都能在终端中操作企业微信”，并明确写出消息、文档、智能表格、待办、会议、日程等能力；但同时保留规模边界：`10 人以上规模企业`强调文档 CLI，`10 人及以下个人/小团队`开放消息、文档、日程、会议、待办等 CLI 能力。来源：[WecomTeam/wecom-cli](https://github.com/WecomTeam/wecom-cli)（访问日期 2026-06-23；GitHub 搜索结果显示仓库最近更新为 2026-05-27，置信度：官方）
- [官方][访问日期 2026-06-23] `wecom-openclaw-plugin` README 继续强调 Bot / Agent 双模式、私聊 / 群聊、流式回复、主动消息、模板卡片，以及“对指定用户、群、部门或标签主动发消息”。这证明 Agent 驱动的企业微信触达能力已很成熟，但发送主体仍是插件接入的机器人 / 应用通道，不应外推为企业微信对所有场景开放。来源：[WecomTeam/wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)（访问日期 2026-06-23；README 兼容说明涉及 2026-03-22 / 2026-03-24 版本分界，置信度：官方）

## 消息发送与 API 开放迹象

- [官方][访问日期 2026-06-23] `发送应用消息` 仍是企业内投递模型，收件目标为 `touser` / `toparty` / `totag`。已确认变化：无。可确认边界：发送主体是自建 / 第三方应用，目标仍是企业内部身份对象。来源：[发送应用消息](https://developer.work.weixin.qq.com/document/path/90236)（置信度：官方）
- [官方][访问日期 2026-06-23] `创建企业群发` 的发送模式并不是“API 直接出站给客户 / 客户群”，而是“创建任务 -> 通知成员 -> 成员确认 -> 发送”。这意味着客户触达依然绑定成员关系、成员执行和操作留痕，不是无成员参与的纯 Agent 广播。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（置信度：官方）
- [官方][访问日期 2026-06-23] 当前官方文档还写着“仅会推送给最后跟客户进行聊天互动的企业成员”。这是权限模型上的强约束，意味着即使创建了企业群发，也不是任意成员都能代发到任意客户 / 客户群。来源：[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)（置信度：官方）
- [官方][2026-05-20] `OpenClaw` 智能机器人帮助页确认了“主动向用户发消息”，但同页又明确“暂不支持在外部群 / 客户群中使用”。因此当前最合理的判断是：`主动消息能力已存在，但目标面和身份面都仍被严格约束在机器人通道内`。来源：[OpenClaw 接入企业微信智能机器人](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21657)（置信度：官方）
- [官方][访问日期 2026-06-23] `wecom-openclaw-plugin` README 中的“对指定用户、群、部门或标签主动发消息”是很强的主动触达信号，但这应被理解为 `Agent / Bot / 应用 outbound channel` 的已实现能力，而不是企业微信对外部联系人、客户群、成员身份代发的全面放开。来源：[WecomTeam/wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)（置信度：官方）
- [推断][基于上述官方文档综合判断] 今天没有看到任何可信新增信号表明：企业微信允许更广泛的 API 或 Agent 直接主动向外部联系人、客户群或任意员工会话发消息。当前最接近的开放，仍然是 `智能机器人 API 模式`、`自建应用消息`、`客户联系群发任务` 三条受控通道。来源同上（置信度：推断，基于官方）

对“发送主体 / 发送目标 / 发送模式 / 权限模型 / 运营限制”的影响拆解如下：

- 发送主体：仍以 `机器人`、`应用`、`成员确认后的群发任务` 为主，没有看到“普通成员身份 API 化代发”的官方新增。
- 发送目标：内部消息可发 `用户 / 部门 / 标签`；客户侧仍是 `客户 / 客户群群发任务`；OpenClaw 机器人官方暂不进入 `外部群 / 客户群`。
- 发送模式：机器人通道已支持被动回复 + 主动消息；客户联系仍偏任务式 / 确认式；未见“任意实时外部会话主动直发”放开。
- 权限模型：仍绑定 `Bot ID / Secret`、应用 Secret、成员关系归属、成员确认、可见范围等受控机制。
- 运营限制：当前官方群发文档口径是“每位客户 / 每个客户群每月最多可接收条数为当月天数”；旧“4 条 / 月”应视为过时二手口径，需停止复用。
