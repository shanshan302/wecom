# 企业微信 AI / Agent 情报日报 - 2026-07-08

## 今日结论

暂无发现 2026-07-08 当天有“企业微信允许 AI Agent 更广泛地直接向外部联系人、客户群或任意会话主动发消息”的官方新证据。

今天的高可信变化仍集中在腾讯云 ADP/ClawPro 与企业微信智能机器人渠道的产品化：ADP 6 月产品动态明确出现“定时任务执行结果推送到微信或企业微信机器人”“远程终端渠道支持企业微信智能机器人、微信 ClawBot 远程对话”；ClawPro 文档也把企微智能机器人渠道做成扫码授权或 Bot ID/Secret 手动配置。结论是：企微正在成为 Agent 的交互入口和执行结果回传入口，但客户联系、外部联系人、客户群群发仍未看到权限模型或运营限制放宽。

## 官方与高可信更新

- [官方][最近更新时间 2026-06-15，检索日期 2026-07-08] 腾讯云 ADP 产品概述显示，ADP 支持 Claw、标准、多智能体、Workflow 四种应用模式，并称智能体 API 提供 RESTful API/SDK，支持发布至企业微信、小程序等常用渠道；同时“连接器与工具生态”全面支持 MCP 协议和自定义扩展。影响：确认企业微信是 ADP 发布渠道之一，MCP/连接器是平台能力，但不等同于企业微信消息 API 放宽。来源：[腾讯云 ADP 产品概述](https://cloud.tencent.com/document/product/1759/104193)
- [官方][最近更新时间 2026-06-11，发布时间 2026-06，检索日期 2026-07-08] 腾讯云 ADP 产品动态列出两条与 WeCom/Agent 相关的 6 月能力：智能工作台定时任务可将执行结果推送到微信或企业微信机器人；远程终端渠道支持通过企业微信智能机器人、微信 ClawBot 远程对话。影响：发送主体更明确是“企业微信机器人/微信 ClawBot”这一渠道，目标是机器人会话或配置渠道，不是客户联系接口的自由外发。来源：[腾讯云 ADP 产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [官方][检索日期 2026-07-08] 腾讯云 ADP《将应用发布到企微智能机器人》说明，企微智能机器人渠道支持长连接 WebSocket 和 URL 回调；长连接由 ADP 服务端与企微建立连接，无需公网域名/IP，使用场景包括群里 @机器人和单聊机器人。影响：已确认机器人会话内的流式回复和消息收发路径；未看到外部联系人或客户群主动触达能力。来源：[将应用发布到企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)
- [官方][最近更新时间 2026-04-15，检索日期 2026-07-08] ADP《将应用发布到企业微信应用》仍是“将 Agent 应用发布至企业微信”的企业应用入口配置，主要通过工作台应用主页/体验链接或定制开发接入。影响：发送主体是企业微信应用入口，不是新增任意消息发送主体。来源：[将应用发布到企业微信应用](https://cloud.tencent.com/document/product/1759/119175)
- [官方][最近更新时间 2026-04-15，检索日期 2026-07-08] ADP《将应用发布到微信客服》要求认证企业准备备案域名，企业微信请求需经 Nginx 代理转发到智能体平台；发布后由微信客服使用智能体回复用户消息。影响：这是微信客服场景的被动/客服回复链路，不构成企微客户群或外部联系人自由主动发送。来源：[将应用发布到微信客服](https://cloud.tencent.com/document/product/1759/122567)
- [半官方/生态][检索日期 2026-07-08] QClaw 企业微信接入文档要求在企业微信中用 API 模式创建智能机器人，选择长连接，获取 Bot ID 和 Secret。影响：进一步确认生态接入的主路径是企微智能机器人长连接，权限边界仍由机器人可使用成员、Bot ID/Secret、企业配置控制。来源：[QClaw 企业微信接入指南](https://qclaw.qq.com/docs/205610395614810112/)

## 消息发送与 API 开放迹象

- [已确认][官方] ADP/ClawPro 可把应用发布到企微智能机器人，支持长连接、扫码绑定或 Bot ID/Secret 配置。发送主体：企微智能机器人或 ADP 应用发布渠道；发送目标：机器人可见范围内的单聊/群聊上下文；发送模式：会话内回复、流式回复、定时任务结果推送；权限模型：企业 ID、Bot ID、Secret、可见成员、发布渠道配置。来源：[ADP 企微智能机器人](https://cloud.tencent.com/document/product/1759/121473)、[ClawPro 一键部署](https://cloud.tencent.com/document/product/1759/128832)
- [已确认][官方] ADP 产品动态中的“定时任务执行结果推送到微信或企业微信机器人”是一个主动结果回传信号，但目标仍是机器人通道。不能推断为可向客户/客户群直接营销触达。来源：[ADP 产品动态](https://cloud.tencent.com/document/product/1759/104191)
- [已确认][官方/镜像] 企业微信客户联系“创建企业群发”模型仍是创建群发任务并通知成员发送给客户或客户群，第三方应用需客户联系群发权限，运营侧仍受成员确认和频控约束。影响：客户/客户群触达仍不是 Agent 直接发送模型。来源：[企业微信开发者中心：创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)、[Apifox 镜像：创建企业群发](https://qiyeweixin.apifox.cn/api-10061297)
- [已确认][官方/生态] 企业微信群机器人 Webhook 仍适合内部群通知，Webhook URL 是关键凭证，不能提交到仓库。影响：发送主体为群机器人，目标通常为内部群；不能外推到外部客户群。来源：[腾讯云：企业微信群机器人接收消息](https://cloud.tencent.com/document/product/1263/71731)
- [推断][半官方/生态] `wecom-openclaw-plugin` 的 proactive messaging 更可能来自 Bot/Agent 双模式、企业内部用户/群/部门/标签、自建应用回退投递，而不是客户联系 API 的新开放。理由：README 同时强调 access control、DM/group policy、Bot-first/Agent-fallback；官方客户联系群发文档没有出现新的 Agent 直发说明。来源：[wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)、[创建企业群发](https://developer.work.weixin.qq.com/document/path/92135)

影响拆解：

- 发送主体：确认有企微智能机器人、企业微信应用、微信客服、群机器人、客户联系群发任务；未确认有“任意 Agent 身份”。
- 发送目标：机器人单聊/群聊、企业内部用户/群/部门/标签、微信客服用户、内部群、客户/客户群群发任务分别属于不同通道。
- 发送模式：机器人通道在强化长连接、流式回复、远程对话、定时任务结果回传；客户联系仍是任务+成员发送。
- 权限模型：Bot ID/Secret、企业 ID、可见范围、备案域名/Nginx、客户联系权限、Webhook key、DM/group allowlist 仍是核心约束。
- 运营限制：未发现客户群、外部联系人频控或成员确认要求被取消。
